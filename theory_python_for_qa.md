# Введение в автоматизацию тестирования

### Как учиться

- Поставьте конкретную цель. Правильная цель — «писать стабильные автотесты на конкретном стеке». Абстрактные формулировки приводят к фрустрации.
- Пишите код постоянно. Уровень автоматизатора определяет объем написанного кода, а не изученной теории. Теоретические знания следует подключать для решения конкретных задач при написании кода.
- Занимайтесь регулярно. Освоение автоматизации — это марафон. Короткие ежедневные сессии эффективнее нерегулярных многочасовых занятий. Ключевой фактор — системность.
- Не бойтесь пересматривать тему с разных сторон. При трудностях с пониманием концепции или инструмента необходимо найти 3-4 альтернативных объяснения. Исходные материалы являются отправной точкой для самостоятельного углубленного поиска.
- Преодолевайте трудности. Объем фреймворков и библиотек для автоматизации велик, но с продвижением освоение упрощается. Практический навык напрямую влияет на интерес: чем стабильнее работают ваши тесты, тем выше удовольствие от процесса.

## Основы автоматизации тестирования

Автоматизированное тестирование представляет собой применение специализированных средств и сценариев для проверки программного обеспечения при минимальном вмешательстве тестировщика. Это позволяет ускорить проверку качества, повысить её эффективность, минимизировать влияние человеческого фактора и гарантировать повторяемость тестовых процедур.
Основные цели автоматизации тестирования:

1. Повышение эффективности: Автоматизированные тесты могут выполняться быстрее и с меньшими ресурсами, чем ручные тесты, особенно при многократной проверке одного и того же функционала.

2. Повторяемость: Скрипты автоматически могут выполняться столько раз, сколько необходимо, с одинаковой точностью и надежностью.

3. Покрытие: Автоматизация позволяет протестировать больше случаев и более сложные сценарии, за более короткое время чем это обычно возможно вручную.

4. Раннее обнаружение дефектов: Запуская автоматические тесты регулярно, можно быстрее выявлять регрессии и дефекты в ранних стадиях разработки.

5. Снижение затрат: Несмотря на начальные инвестиции в создание тестов, автоматизация может значительно сократить затраты на тестирование в долгосрочной перспективе


### Когда автоматизация тестов не нужна:

1. Тесты нового функционала, который быстро меняется:
   - На ранних стадиях разработки, когда требования и функционал сильно меняются, может быть нецелесообразно вкладывать ресурсы в автоматизацию, которая быстро устареет.

2. Тесты, требующие субъективной оценки:
   - Дизайн, удобство использования и другие аспекты, более подходящие для человеческой оценки.

3. Однократные или редко повторяющиеся тесты:
   - Если тестовый сценарий будет выполнен всего один раз или очень редко, его автоматизация может быть не оправдана.

4. Время на разработку превышает стоимость ручного выполнения:
   - Если написание автоматизируемого теста требует больше времени и ресурсов, чем выполнение его вручную несколько раз, автоматизация может быть неэффективной.

Рекомендации:

- Четко определите ROI (возврат инвестиций): Оцените затраты времени и ресурсов на автоматизацию в сравнении с потенциальной выгодой.
- Начинайте с самого критичного и повторяемого: Функционал с высоким риском и часто повторяемые тесты — отличная отправная точка.
- Используйте гибридный подход: Смешивайте автоматизацию и ручное тестирование для достижения наилучших результатов.

В конечном итоге, решение об автоматизации тестов должно быть основано на балансе между затратами и ожидаемыми выгодами, учитывая характер проекта и его требования.

### Когда следует автоматизировать тесты:

Автоматизация тестирования может предложить значительные преимущества, но не все тесты стоит автоматизировать. Рассмотрим, когда автоматизация особенно оправдана:

1. Регрессионное тестирование:
   - Часто повторяющиеся тесты, которые проверяют, не возникли ли новые дефекты после изменений в коде, особенно в крупных и долгосрочных проектах.
   - Автоматизация здесь может существенно сэкономить время и уменьшить вероятность упущения регрессионных дефектов.

2. Стабильные части системы:
   - Стабильные функциональные элементы, которые редко меняются и которым необходимо обеспечить постоянное качество.

3. Тесты, требующие большого объема данных или выполняемые на нескольких конфигурациях:
   - Тесты, которые могут включать различные наборы данных или различные среды (например, разные операционные системы или версии браузеров).

4. Тесты производительности и нагрузки:
   - Сценарии, требующие поведения приложения под нагрузкой, замеряющие его производительность и устойчивость.

5. Тесты, которые должны запускаться часто:
   - Любые тесты, которые запускаются на регулярной основе, например, ночные сборки или CI/CD пайплайны.

6. Сложные сценарии, подверженные человеческим ошибкам при ручном тестировании:
   - Тесты, где большое количество шагов или сложные вычисления делают процесс слишком трудным для выполнения вручную без ошибок.

## Переменные

В Python переменные используются для хранения данных. Они создаются просто присваиванием значения имени переменной, без необходимости явно указывать её тип. Python автоматически определяет тип переменной на основе присваиваемого значения.

Можно создать переменную, просто присвоив значение:

```python
count = 10
name = "Иван"
price = 19.99
```

Выбор имен и использование переменных в Python:

Когда вы работаете с переменными в Python, стоит следовать ряду правил и рекомендаций для создания более понятного кода.

- Имена переменных должны содержать только буквы, цифры и символы подчеркивания (_). Они начинаются с буквы или символа подчеркивания, например, `message_1`, но не с цифры, например, `1_message` неверно.
  
- Пробелы в именах переменных недопустимы. Для разделения слов используйте символы подчеркивания, например, `greeting_message`, но не `greeting message`.

- Не используйте имена функций и ключевые слова Python в качестве имен переменных. Например, слово `print` зарезервировано.

- Имена переменных должны быть краткими и информативными. Например, name лучше, чем n; student_name лучше, чем s_n; name_length предпочтительнее length_of_persons_name.

- Будьте внимательны при использовании строчной буквы `l` и прописной `O`, так как они похожи на цифры `1` и `0`.

С опытом ваши имена переменных станут более понятными и осмысленными.

## Строки

В Python строки — это последовательности символов, которые используются для хранения и работы с текстовой информацией. 
1. Создание строк:

Строки можно создавать, заключая текст в одинарные ('...') или двойные кавычки ("..."). Оба варианта работают одинаково.

```python
single_quote = 'Пример строки'
double_quote = "Пример строки"
```

Это позволяет использовать внутренние кавычки и апострофы в строках:

```python
string_with_internal_quotation_marks = 'Пример "Строки" с внутренними кавычками'
```

2. Многострочные строки:

Для многострочных строк используются тройные кавычки (как одинарные, так и двойные).

```python
multi_line = '''Это
   многострочная
   строка'''
```

3. Экранирование символов:

Символы, которые имеют специальное значение, можно экранировать с помощью обратной косой черты (\).

```python
escaped_single_quote = 'It\'s a string'
new_line = "Строка с\nновой строкой"
```

4. Операции со строками:


   - Конкатенация: Склеивание строк с помощью оператора +.

```python
greeting = "Hello" + " " + "world"
```


   - Повторение: Умножение строки на число дублирует её.

```python
repeated = "echo " * 3
```

5. Доступ к символам строки:

Доступ к символам можно получить с помощью индексов. Индексы начинаются с 0.

```python
word = "Python"
   first_letter = word[0]  # 'P'
   last_letter = word[-1]  # 'n'
```

6. Срезы:

Срезы в Python позволяют извлекать подстроки из строк, а также создавать новые списки из существующих. Синтаксис срезов включает указание начального и конечного индексов, а также шага. Формат записи среза: sequence[start:stop:step].

Вот более детальное объяснение и примеры:

1. Основной синтаксис: 
   - start: начальный индекс (включительно), с которого начинается срез. По умолчанию — 0.
   - stop: конечный индекс (не включается в срез), на котором срез заканчивается. По умолчанию — длина последовательности.
   - step: шаг, определяет, какой элемент будет следующим включен в срез. По умолчанию — 1.

2. Примеры использования:

```python
# Строка для примеров
text = "Python Programming"

# Извлечение с начала до индекса 6 (6 не включается)
slice1 = text[0:6]
print(slice1)  # Output: "Python"

# Извлечение с индекса 7 до конца
slice2 = text[7:]
print(slice2)  # Output: "Programming"

# Извлечение с начала до индекса 6 с шагом 2
slice3 = text[0:6:2]
print(slice3)  # Output: "Pto"

# Извлечение всей строки (аналог копирования)
slice4 = text[:]
print(slice4)  # Output: "Python Programming"

# Извлечение с конца (реверс строки)
reversed_text = text[::-1]
print(reversed_text)  # Output: "gnimmargorP nohtyP"

# Извлечение каждого второго символа во всей строке
slice5 = text[::2]
print(slice5)  # Output: "Pto rgamn"
```

3. Особенности:
   - Если start опущен, срез начинается с начала последовательности.
   - Если stop опущен, срез продолжается до конца.
   - Если step отрицательный, срез производится в обратном порядке.

Срезы являются чрезвычайно мощным инструментом для манипуляции строками, списками и другими последовательностями в Python
   
7. Длина строки:

Для получения длины строки используется функция len().

```python
length = len(word)  # 6
```

8. Методы строк:

Строки обладают множеством встроенных методов, таких как .upper(), .lower(), .strip(), .replace(), .split(), и другие.

upper() - Преобразует все символы строки в заглавные буквы.

```python
text = "Hello, World!"
upper_text = text.upper()
print(upper_text)  # "HELLO, WORLD!"
```

lower() - Преобразует все символы строки в строчные буквы.

```python
text = "Hello, World!"
lower_text = text.lower()
print(lower_text)  # "hello, world!"
```

strip() - Удаляет пробелы (или другие указанные символы) в начале и конце строки.

```python
text = "   Hello, World!   "
stripped_text = text.strip()
print(stripped_text)  # "Hello, World!"
```

replace(old, new) - Заменяет все вхождения подстроки old на подстроку new.

```python
text = "Hello, World!"
replaced_text = text.replace("World", "Python")
print(replaced_text)  # "Hello, Python!"
```

split(sep=None) - Разделяет строку на список подстрок по указанному разделителю. Если разделитель не указан, строка разбивается по пробелам.

```python
text = "Hello, World!"
split_text = text.split(", ")
print(split_text)  # ["Hello", "World!"]
```

```python
# Пример разделения по пробелам
sentence = "This is a test."
words = sentence.split()
print(words)  # ['This', 'is', 'a', 'test.']
```

Эти методы позволяют легко манипулировать строками и извлекать нужную информацию из текстовых данных в Python.

9. Неизменяемость:

Строки в Python неизменяемы, то есть после создания их нельзя изменить. Любые операции, которые "изменяют" строку, на самом деле создают новую строку.

Строки в Python очень полезны и широко используются для работы с текстом, а их методы и операции позволяют с легкостью манипулировать текстовыми данными.

## Использование переменных в строках

f-строки — это наиболее удобный и современный метод для вставки переменных и выражений в строки. Для создания f-строки перед строкой ставится буква f, а переменные вставляются в фигурные скобки {}.

```python
name = "Alice"
age = 30
message = f"Привет, {name}! Тебе {age} лет."
print(message)
```

F-строки также поддерживают более сложные выражения и вызовы функций:

```python
height = 1.7
message = f"{name} ростом {height * 100:.1f} см."  # Результат: "Alice ростом 170.0 см."
print(message)
```

F-строки являются наиболее предпочтительным и удобным способом встраивания переменных в строки в современном Python благодаря их простоте и читабельности.

## Числа

В Python существует несколько типов чисел, которые используются для различных математических операций и представления данных. Вот основные числовые типы:
1. Целые числа (int)

Целые числа представляют собой числа без десятичной точки. В Python int может быть любого размера — ограничение накладывается только доступной памятью.

```python
a = 42  # Пример целого числа
b = -7  # Отрицательное целое число
```

2. Вещественные числа (float)

Вещественные числа представляют собой числа с плавающей запятой, то есть с десятичной точкой. Этот тип данных используется для представления чисел с дробной частью.

```python
x = 3.14159  # Пример вещественного числа
y = -0.001
```

Операции с числами

Python поддерживает различные операции с числами, такие как сложение, вычитание, умножение, деление и так далее:

- Сложение: a + b
- Вычитание: a - b
- Умножение: a * b
- Деление: a / b (всегда возвращает float)
- Целочисленное деление: a // b (возвращает целую часть от деления)
- Остаток от деления: a % b
- Возведение в степень: a ** b

 
## Комментарии

Цель: Научиться писать эффективные комментарии для улучшения читаемости кода тестов.
1. Зачем нужны комментарии?

- Объясняют логику: Помогают понять сложные участки кода.
- Экономят время: Позволяют быстро разобраться в коде после перерыва или при работе в команде.
- Документируют решения: Поясняют, почему выбран конкретный подход.

💡 *В тестировании комментарии особенно важны для описания:

- Тестовых сценариев
- Причины использования конкретных проверок
- Работы с неочевидными элементами системы*

2. Синтаксис комментариев

В Python комментарии начинаются с символа #. Всё после этого символа игнорируется интерпретатором.

Пример 1: Простой комментарий

```python
# Проверка успешной авторизации  
def test_login_success():  
    # Вызов метода ввода логина/пароля  
    enter_credentials("admin", "secret")  
    assert is_welcome_page_displayed()  # Проверка перехода на главный экран
```

Пример 2: Закомментированный код

```python
def test_data_processing():  
    # old_data = fetch_data_from_v1_api()  # Старая версия API, не поддерживается  
    new_data = fetch_data_from_v2_api()  
    assert new_data.is_valid()
```

Хорошие комментарии:

Объясняют "почему":

```python
# Используем ожидание 10 сек из-за медленной загрузки 3D-модели
```

```python
# Проверяем через хэш SHA-256, т.к. API возвращает данные в зашифрованном виде  
assert hashlib.sha256(user_data.encode()).hexdigest() == stored_hash
```

Указывают на временные решения:

```python
# TODO: Заменить на get_token() после исправления бага сервера #AUTH-123  
auth_token = "temp_token_123"
```

### Многострочные комментарии

Цель: Научиться оформлять многострочные комментарии для документирования сложной логики в тестах.
1. Способы создания многострочных комментариев

В Python нет специального синтаксиса для многострочных комментариев, но используются:

- Серия однострочных комментариев
- Строковые литералы в тройных кавычках (технически — не комментарии, но могут выполнять их роль)

```python
# Это предпочтительный способ
# для многострочных комментариев.
# Каждая строка начинается с решетки.

"""
Это строковый литерал, а не истинный комментарий.
Интерпретатор создает объект строки, но если 
он ничему не присвоен — игнорируется.
Используйте осторожно!
"""
```

2. Когда использовать многострочные комментарии

- Документирование сложной бизнес-логики
- Описание архитектурных решений в тестах
- Временное отключение блоков кода при отладке

⚠️ Важно! Тройные кавычки — это не комментарии, а строки. Они могут создавать неявные объекты в памяти.

- Для пояснений в коде: всегда используйте #
- Для временного кода: тройные кавычки (с пометкой TODO)
- Для документирования: docstrings (""" в начале функции)

Избегайте:


- Всегда обновляйте комментарии при изменении кода!

## Дзен Python

Опытные программисты Python рекомендуют избегать лишних сложностей и по возможности выбирать простые решения там, где это возможно. Приведу несколько строк, чтобы вы поняли, почему они важны для вас как для начинающего программиста Python.

- Красивое лучше, чем уродливое.

Код может быть красивым и элегантным. Короткие и понятные имена переменных, названий методов и функций, четкая и понятная структура. Код должен быть написан так, чтобы его легко было понять другому разработчику (или вам самим через несколько месяцев).

Не стоит пытаться уместить всё в одну строку только ради краткости — зачастую это делает код нечитаемым и трудно поддерживаемым.

«Красивый» код — это не только про внешний вид, но и про ясность намерений, осмысленные имена переменных и простую логику.

- Простое лучше, чем сложное.

Если у вас есть выбор между простым и сложным решением и оба работают, используйте простое решение. Код будет проще в сопровождении, а у вас и других разработчиков будет меньше проблем с обновлением этого кода в будущем.

Сложное лучше, чем запутанное.

Реальность создает свои сложности; иногда простое решение задачи невозможно. В таком случае используйте самое простое решение, которое работает.

Удобочитаемость имеет значение.

Даже если ваш код сложен, постарайтесь сделать так, чтобы он нормально читался. Работая над проектом, требующим написания сложного кода, постарайтесь написать содержательные комментарии для этого кода.

Должен существовать один — и желательно только один — очевидный способ сделать это.

Если предложить двум программистам Python решить одну и ту же задачу, они должны выработать похожие решения. Это не значит, что в программировании нет места для творчества. Наоборот! 
Но большая часть работы программиста заключается в применении небольших стандартных решений для простых ситуаций в контексте большого, более творческого проекта. 
Внутренняя организация ваших программ должна выглядеть логично с точки зрения других программистов Python.

Сейчас лучше, чем никогда.

Вы можете потратить весь остаток жизни на изучение всех тонкостей Python и программирования в целом, но в таком случае вы никогда не закончите ни одного проекта. 
Не пытайтесь написать идеальный код; напишите код, который работает, а потом решите, стоит ли доработать его для текущего проекта или перейти на что-то другое. 
Когда вы перейдете к следующей главе и займетесь изучением более сложных тем, постарайтесь не забывать об этой философии простоты и ясности.
Опытные программисты будут с большим уважением относиться к вашему коду, более охотно поделятся своим мнением и скорее будут сотрудничать с вами в интересных проектах.

## Списки

Списки в Python — это один из наиболее часто используемых типов данных. Они позволяют хранить коллекции элементов, которые могут быть любыми объектами: числами, строками, даже другими списками. Давайте рассмотрим основные моменты, которые важно знать новичку о списках, с простыми пояснениями и примерами.
Создание списка

Список создается с помощью квадратных скобок [], и элементы разделяются запятыми.

```python
# Создаём пустой список
empty_list = []
```

```python
# Список с элементами
fruits = ["apple", "banana", "cherry"]
```

Доступ к элементам

Элементы списка индексируются, начиная с 0, а доступ к ним можно получить с помощью квадратных скобок.

```python
print(fruits[0])  # выводит "apple"
print(fruits[1])  # выводит "banana"
```

Изменение элементов

Списки в Python изменяемы, что означает, что вы можете изменять их содержимое.

```python
fruits[0] = "orange"
print(fruits)  # выводит ["orange", "banana", "cherry"]
```

Добавление элементов

Для добавления элементов в список используются методы append() и insert().

```python
# Добавить элемент в конец списка
fruits.append("mango")  
print(fruits)  # выводит ["orange", "banana", "cherry", "mango"]

# Вставить элемент на определённую позицию
fruits.insert(1, "strawberry")  
print(fruits)  # выводит ["orange", "strawberry", "banana", "cherry", "mango"]
```

Удаление элементов

Для удаления элементов можно использовать методы remove(), pop() или оператор del.

```python
# Удалить элемент по значению
fruits.remove("banana")
print(fruits)  # выводит ["orange", "strawberry", "cherry", "mango"]

# Удалить элемент по индексу и получить его
popped_fruit = fruits.pop(2)  
print(popped_fruit)  # выводит "cherry"
print(fruits)  # выводит ["orange", "strawberry", "mango"]

# Удалить элемент по индексу с использованием del
del fruits[0]  
print(fruits)  # выводит ["strawberry", "mango"]
```

Итерация по списку

Вы можете перебрать элементы списка с помощью цикла for.

```python
for fruit in fruits:
    print(fruit)
```

Длина списка

Чтобы узнать, сколько элементов в списке, используйте len().

```python
num_fruits = len(fruits)
print(num_fruits)  # выводит 2
```

Другие полезные методы

- sort() - сортирует список.
- reverse() - изменяет порядок элементов списка на обратный.
- count() - возвращает количество вхождений элемента в список.
- index() - возвращает индекс первого вхождения элемента.

numbers = [3, 1, 4, 1, 5, 9, 2]

```python
# Сортировка
numbers.sort()
print(numbers)  # выводит [1, 1, 2, 3, 4, 5, 9]

# Обратный порядок
numbers.reverse()
print(numbers)  # выводит [9, 5, 4, 3, 2, 1, 1]

# Количество вхождений
count_of_ones = numbers.count(1)
print(count_of_ones)  # выводит 2

# Первый индекс вхождения
index_of_four = numbers.index(4)
print(index_of_four)  # выводит 2
```

Списки — это мощный и гибкий инструмент в Python, который делает работу с набором данных удобной и эффективной. Они позволяют легко изменять, добавлять и удалять элементы, а также предлагают множество встроенных методов для часто встречающихся задач.

## Условные операторы

Условные операторы в Python позволяют вашему коду принимать решения в зависимости от выполнения тех или иных условий. Это один из ключевых компонентов, который поможет вам создать программы, способные реагировать на разные ситуации. Давайте рассмотрим основные моменты об условных операторах, которые нужно знать новичку, с простыми объяснениями и примерами.

Основные условные операторы

1. if - ключевое слово, которое используется для проверки условия. Если условие истинно (True), выполняется блок кода под ним.

2. elif (сокращение от "else if") - позволяет проверить дополнительные условия, если предыдущие условия не были истинными.

3. else - определяет блок кода, который выполняется, если ни одно из предыдущих условий не было истинным.
Синтаксис условных операторов

```python
if условие:
    # код выполняется, если условие истинно
elif другое_условие:
    # код выполняется, если другое_условие истинно
else:
    # код выполняется, если ни одно из условий не истина
```

Пример использования

Пример: программа, определяющая, является ли число положительным, отрицательным или нулём.

```python
number = 10

if number > 0:
    print("Число положительное")
elif number < 0:
    print("Число отрицательное")
else:
    print("Число равно нулю")
```

Составные условия

Вы можете использовать логические операторы and, or и not для комбинирования нескольких условий.

- and - истина только тогда, когда оба условия истинны.
- or - истина, если хотя бы одно из условий истинно.
- not - инвертирует значение условия.

Пример с логическими операторами:

```python
age = 20
has_license = True

if age >= 18 and has_license:
    print("Можно водить машину")
else:
    print("Водить машину нельзя")
```

Вложенные условия

Вы можете вкладывать одни условные операторы в другие для проверки более сложных условий.

```python
x = 5

if x > 0:
    if x < 10:
        print("x - положительное число меньше 10")
    else:
        print("x - положительное число больше или равно 10")
else:
    print("x - неположительное число")
```

Советы для работы с условными операторами

Аккуратное форматирование:

Используйте отступы (обычно 4 пробела) для указания уровней вложенности. Это делает код читабельным и легким для понимания.

Понятные условия:

Пишите условия, которые легко понять. Если они становятся слишком сложными, подумайте об их упрощении.

Порядок elif и else:

Размещайте условия слева направо по убывающей вероятности их истинности, чтобы быть уверенным, что сравнения выполняются в том порядке, в котором нужно.

Условные операторы превращают ваш код в более умный и адаптивный, позволяя выполнять различные действия в зависимости от входных данных и текущего состояния программы. Это — основа создания динамических программ.

## Словари

Словари в Python — это удобная структура данных, которая позволяет хранить и управлять данными в виде пар "ключ-значение". Они похожи на адресные книги, где каждому имени (ключу) соответствует номер телефона (значение). Вот что нужно знать новичку о словарях с простыми объяснениями и примерами:
Что такое словарь?

- Словарь — это неупорядоченная коллекция пар "ключ-значение".
- Ключи должны быть уникальными и неизменяемыми данными (например, строки, числа, кортежи).
- Значения могут быть любого типа данных и могут дублироваться.
Создание словаря

Можно создать пустой словарь или сразу заполнить его значениями:

```python
# Пустой словарь
empty_dict = {}

# Словарь с начальными значениями
person = {
    "name": "Alice",
    "age": 30,
    "city": "New York"
}
```

Доступ к значениям

Доступ к значению в словаре осуществляется по ключу, используя квадратные скобки [].

```python
print(person["name"])  # выводит "Alice"
print(person["age"])   # выводит 30
```

Изменение и добавление элементов

Вы можете добавлять новые пары "ключ-значение" или изменять значения существующих ключей.

```python
# Изменение значения
person["age"] = 31

# Добавление нового ключа-значения
person["email"] = "alice@example.com"
```

Удаление элементов

Можно удалить элемент словаря по ключу с помощью del или метода pop().

```python
# Удаление с помощью del
del person["city"]

# Удаление с помощью pop (и получение удалённого значения)
email = person.pop("email")
```

Проверка на наличие ключа

Перед доступом к значению можно проверить существование ключа с помощью оператора in.

```python
if "name" in person:
    print("Имя:", person["name"])
```

Перебор ключей и значений

Словари можно перебирать несколькими способами.

- Перебор ключей:

```python
for key in person:
    print(key, ":", person[key])
```

- Перебор значений:

```python
for value in person.values():
    print(value)
```


- Перебор пар ключ-значение:

```python
for key, value in person.items():
    print(key, ":", value)
```

Длина словаря

Чтобы узнать количество элементов в словаре, используйте функцию len().

```python
num_items = len(person)
print(num_items)  # выводит 2, если остались только "name" и "age"
```

Примеры использования

Словари незаменимы, когда нужно быстро искать данные по ключу. Например, они удобны для хранения данных конфигурации, определения ассоциаций или работы с JSON-данными, где ключи и значения соответствуют структуре данных.

Ниже приведен пример использования словаря:

```python
# Создание словаря
phone_book = {
    "Иван": "+7-123-456-7890",
    "Мария": "+7-987-654-3210",
    "Алексей": "+7-555-555-5555"
}

# Добавление новой записи в словарь
phone_book["Светлана"] = "+7-333-333-3333"

# Изменение существующей записи
phone_book["Иван"] = "+7-999-999-9999"

# Удаление записи
del phone_book["Алексей"]

# Поиск телефонного номера по имени
name = "Мария"
phone = phone_book.get(name, "Контакт не найден")
print(f"Телефон {имя}: {phone}")

# Перебор всех элементов словаря
for name, phone in phone_book.items():
    print(f"{name}: {phone}")

# Проверка наличия ключа в словаре
if "Светлана" in phone_book:
    print("Светлана есть в телефонной книге")

# Получение всех ключей и значений
all_names = phone_book.keys()
all_phones = phone_book.values()
```

Словари — это гибкий инструмент для организации данных, особенно когда нужно быстро находить и изменять значения по уникальным ключам. Научившись работать со словарями, вы сможете более эффективно решать задачи, связанные с хранением и обработкой структурированных данных.

## Ввод данных

В Python для ввода данных с клавиатуры используется функция input(). Она принимает необязательный строковый аргумент, который выводится пользователю как подсказка. Введенные пользователем данные всегда возвращаются как строка. Если вам нужно число, то строку нужно преобразовать в нужный тип (например, int для целых чисел).

Пример:

```python
# Ввод имени пользователя
name = input("Введите ваше имя: ")
print(f"Привет, {name}!")
```

```python
# Ввод числа и преобразование в целое число
age = input("Введите ваш возраст: ")
age = int(age)
print(f"Через год вам будет {age + 1} лет.")
```

## Циклы

В Python существуют два основных вида циклов: for и while.

### Цикл for

Цикл for используется для перебора элементов последовательностей (например, списков, строк или диапазонов чисел).

Пример перебора списка:

```python
fructs = ["яблоко", "банан", "вишня"]
for fruct в fructs:
    print(fruct)
```

Функция range() генерирует последовательность чисел, которую можно использовать в цикле for.

```python
# Этот цикл выведет числа от 1 до 5.
for number in range(1, 6):
    print(f"Число: {number}")
```

Перебор словаря:

Можно перебирать ключи, значения или пары "ключ-значение" в словаре.

```python
capitals = {
    "Россия": "Москва",
    "Франция": "Париж",
    "Япония": "Токио"
}

# Перебор ключей
for capital in capitals:
    print(f"Страна: {capital}")

# Перебор значений
for city in capitals.values():
    print(f"Столица: {city}")

# Перебор пар "ключ-значение"
for страна, город in столицы.items():
    print(f"Столица {страна} — это {город}")
``` 

### Цикл while

Цикл while выполняется до тех пор, пока условие истинно. Этот цикл полезен, когда количество повторений заранее неизвестно и зависит от какого-то условия.

Пример использования цикла while:

```python
number = 0
while number < 5:
    print(number)
    number += 1
```

В этом примере цикл продолжается, пока переменная число меньше 5.
Важные моменты

- Бесконечные циклы: Будьте осторожны с циклами while, чтобы случайно не создать бесконечный цикл. Это может случиться, если условие всегда остается истинным.
  
- Прерывание цикла: В цикле можно использовать break для его прерывания досрочно и continue для перехода к следующей итерации.

Пример использования break и continue:

```python
for number in range(10):
    if number == 5:
        break  # Остановит цикл полностью
    if number % 2 == 0:
        continue  # Перейти к следующей итерации, пропуская то, что ниже
    print(f"Нечетное число: {number}")
```

## Функции

Функция — это блок кода, которому можно дать имя и который можно использовать многократно. Функции помогают упростить и организовать код, избегая повторений. 
Как определить функцию?

Чтобы создать (определить) функцию в Python, используется ключевое слово def, за которым следует имя функции, круглые скобки и двоеточие. После этого идет блок кода, который относится к функции и записывается с отступом.

Вот простой пример:

```python
def hello_world():
    print("Привет, мир!")
```

Этот код определяет функцию с именем приветствие, которая выводит на экран строку "Привет, мир!".
Как вызвать функцию?

Чтобы использовать функцию, достаточно указать ее имя и круглые скобки:

```python
hello_world()  # вызов функции
```

При вызове этой функции на экран будет напечатано: "Привет, мир!".
Параметры функции

Функции могут принимать входные данные, называемые параметрами или аргументами. Параметры указываются в круглых скобках при определении функции.

Вот пример функции с одним параметром:

```python
def greeting_with_name(name):
    print(f"Привет, {name}!")

greeting_with_name("Алиса")  # Выведет: Привет, Алиса!
greeting_with_name("Боб")   # Выведет: Привет, Боб!
```

Здесь функция приветствие_с_именем принимает один параметр имя, который используется внутри функции.
Возврат значений

Функции могут возвращать значения с помощью ключевого слова return. Это позволяет функции вычислять результат и передавать его обратно.

Пример функции, возвращающей сумму двух чисел:

```python
def summ(a, b):
    return a + b

result= summ(3, 4)
print(result)  # Выведет: 7
```

Именованные и необязательные параметры

В Python можно задавать значения по умолчанию для параметров. Это делается путем указания значения после знака = в определении функции.

```python
def greeting(name, message="Как дела?"):
    print(f"Привет, {name}! {message}")

greeting("Алиса")                 # Выведет: Привет, Алиса! Как дела?
greeting("Боб", "Что нового?")   # Выведет: Привет, Боб! Что нового?
```

Здесь параметр сообщение имеет значение по умолчанию "Как дела?". Его можно изменить при вызове функции.

### Основные моменты

1. Определение функций: с помощью def.
2. Вызов функций: по имени функции с параметрами в скобках.
3. Параметры: дают возможность функции использовать разные входные данные.
4. Возврат значений: с помощью return.
5. Значения по умолчанию: позволяют делать параметры необязательными.

Использование функций делает ваш код более структурированным и читаемым, облегчает повторное использование и тестирование отдельных частей программы.
