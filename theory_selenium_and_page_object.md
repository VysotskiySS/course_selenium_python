# Тестирование WEB с помощью Selenium
 
### Что такое веб-страница?

В этом уроке мы детально рассмотрим, как работать с элементами веб-страницы с помощью Selenium WebDriver и как обращаться к конкретному элементу страницы с помощью CSS и XPATH. Каждая страница в интернете — это, по сути, HTML-файл, где с помощью языка разметки HTML описана структура страницы.

Помимо HTML, важно знать о CSS (Cascading Style Sheets), который отвечает за стиль и оформление страниц, делая их привлекательными и удобными в использовании. 

Пример CSS-кода:

```css
button {
  color: green;
}
```

Здесь button является селектором, который выбирает все элементы <button> на странице, а color: green; задает им зеленый цвет.
Почему важна структура страницы?

Для автоматизации тестирования нам важно знать, как устроена страница, то есть как она описана на языке HTML. Понимание структуры помогает определять путь к элементам страницы. Это позволяет находить нужные элементы и выполнять с ними действия: например, вводить текст в поле или нажимать кнопки.

HTML-страницы имеют древовидную структуру, где элементы могут содержать вложенные элементы, создавая иерархию.
Иерархия в HTML-документах

HTML-документ можно сравнить с семейным древом: у элементов есть родители, дети, братья и предки. Это потому, что элементы могут быть вложены друг в друга. Дочерние элементы могут сами иметь свои дочерние элементы, что позволяет создавать сложные структуры.
Термины иерархии:

- Потомок элемента X: Любой элемент, находящийся внутри элемента X, независимо от уровня вложенности.
- Ребёнок (дочерний элемент): Прямой потомок элемента, то есть элемент, находящийся на первом уровне вложенности внутри родителя.
- Предок элемента Y: Любой элемент, который содержит элемент Y.
- Родитель: Прямой предок, тот, кто находится ровно на один уровень выше.
- Братский (соседний) элемент: Элемент на том же уровне, что и другой элемент, с общим родителем.

Пример HTML-кода:

```html
<section class="article">
  <header class="header">
    <h1 class="header-title">Новости о Python</h1>
  </header>
  <article class="content">
    <p class="paragraph-one">Python 3.9 выпущен с множеством новых функций.</p>
    <p class="paragraph-two">Вы можете загрузить новую версию Python 3.9 и попробовать его возможности.</p>
  </article>
</section>
```


Разбор:

- `<section class="article">` — это родитель для `<header class="header">` и `<article class="content">`.
- `<h1 class="header-title">` — это ребёнок `<header class="header">`, а также потомок `<section class="article">`.
- `<p class="paragraph-one">` и `<p class="paragraph-two">` — соседи, так как они на одном уровне и находятся внутри общего родителя `<article class="content">`.


### Как искать элементы на веб-странице?

Существует несколько методов поиска элементов внутри страницы:

1. По значениям тегов или атрибутов: Можно находить элементы по уникальным значениям их ID, классов и других атрибутов.

2. CSS-селекторы: Используются, чтобы описывать путь к элементу с помощью синтаксиса CSS. CSS-селектор подобен инструкции, которая указывает на конкретное место на странице.

3. XPath: Использует язык запросов для описания пути к элементам в виде выражений.

Эти способы помогут вам эффективно находить нужные элементы и взаимодействовать с ними при автоматизации тестов. В следующих уроках мы более подробно рассмотрим каждый метод и научимся применять их на практике.

### Теги и атрибуты

1. Теги HTML: Каждый элемент на веб-странице описан с помощью HTML-тегов, таких как `<div>`, `<input>`, `<button>`, `<a>` и т.д. Знание того, какие теги используются для различных элементов, поможет вам найти нужные части интерфейса.
Например, `<h1>` обозначает заголовок, `<p>` — абзац, `<a>` — гиперссылка.

Пример:

```html
<button>Кликни меня</button>
```

это кнопка с тегом `<button>`.

2. Атрибуты элементов: Каждый элемент может иметь несколько атрибутов, которые дают ему дополнительные характеристики. Одни из самых важных атрибутов — это id и class:
   - ID: Этот атрибут обычно уникален для элемента на странице. Если элемент имеет атрибут id, вы можете использовать его для поиска конкретного элемента.

Пример:

```html
<input id="username" type="text">
```

здесь id равен "username", что позволяет легко найти это текстовое поле.

3. Атрибут Class: Элементы могут иметь один или несколько классов, которые помогают их стилизовать. Классы не обязаны быть уникальными, поэтому могут применяться ко многим элементам. Он может помочь в поиске и группировке элементов.

Пример:

```html
<div class="menu-item">Меню</div>
```

class равен "menu-item". Этот же класс может быть у нескольких элементов.

4. Другие атрибуты:
   - Элементы могут иметь различные атрибуты, такие как name, type и специальные атрибуты data-* для хранения дополнительных данных.

Пример:

```html
<input name="email" type="email" data-info="user-email">
```

здесь мы можем использовать атрибуты name или data-info для поиска элемента.

5. Комбинирование критериев поиска: Можно комбинировать теги и атрибуты, чтобы более точно найти элемент. Например, искать <input> элемент с определённым id или классом.
   - Можно использовать комбинации тегов и атрибутов для более точного поиска.

Пример:

```css
input[type="text"].important
```

это пример CSS-селектора, который находит текстовые поля с классом "important".

6. Инструменты браузера:
   - В браузере есть инструменты разработчика (обычно открываются клавишей F12), которые помогут вам увидеть HTML-код страницы. С их помощью можно наводить мышкой на элементы и сразу видеть их теги и атрибуты.

### CSS-селекторы

Что такое CSS-селекторы?

CSS-селекторы — это формулы или шаблоны, которые позволяют вам находить элементы на веб-странице на основе атрибутов, классов, ID и структуры DOM (Document Object Model). Они широко используются для стилизации, но также отлично подходят для написания тестов.

Основные виды CSS-селекторов

1. Селектор по тегу: выбирает все элементы данного тега.

`div`

Выбирает все <div> элементы.

2. Селектор по классу: выбирает элементы с указанным классом.

`.classname`

Выбирает все элементы с классом classname.

3. Селектор по ID: выбирает элемент с указанным ID.

`#unique-id`

Выбирает элемент с ID unique-id.

4. Комбинированные селекторы: выбирают элементы, которые соответствуют всем указанным условиям.

`div.myClass`

Выбирает все <div> элементы с классом myClass.

5. Групповые селекторы: позволяют выбрать несколько разных элементов.

`div, p, a`

Выбирает все `<div>`, `<p>`, и `<a>` элементы.
Комбинаторы

- Потомок (  (пробел)): выбирает элементы внутри других элементов.

`div p`

Все `<p>` внутри `<div>`.

- Дочерний (`>`): выбирает элементы, которые являются непосредственными потомками.

`div > p`

Все `<p>`, которые непосредственно внутри `<div>`.

- Сосед справа (`+`): выбирает элемент, который расположен непосредственно после другого.

`h1 + p`

Выбирает `<p>`, который сразу после `<h1>`.

- Сосед с общей родительской оберткой (`~`): выбирает все элементы, которые находятся за определенным элементом с общей оберткой.

`h1 ~ p`

Выбирает все <p>, следующие за `<h1>`, имеющие общего родителя.
Специальные селекторы

- Селектор атрибута: выбирает элементы с определённым атрибутом.

`input[type="text"]`

Выбирает все `<input>`, у которых type равен text.

- Селекторы псевдоклассов:
  - `:first-child`: выбирает первый дочерний элемент в родительском элементе.

`p:first-child`

Выбирает первый `<p>` в каждом родителе.
  
  - `:last-child`: выбирает последний дочерний элемент.

`p:last-child`

Выбирает последний `<p>` в каждом родителе.

  - `:nth-child(n)`: выбирает n-й дочерний элемент.

`li:nth-child(2)`

Выбирает второй `<li>` элемент в каждом родителе.
Примеры использования CSS-селекторов

- Выбор всех кнопок с определённым классом:

`button.login`

 - Выбор первого параграфа внутри каждого `div`:

`div p:first-child`

- Выбор всех элементов с атрибутом `data-test`:

`[data-test]`

- Выбор ссылок, у которых `href` начинается с "http":

`a[href^="http"]`

 Полезные рекомендации

- Проверяйте селекторы в DevTools (Ctrl+Shift+C в браузере)
- Избегайте избыточных селекторов: .header nav ul li → .header li
- Оптимальный путь: классы > атрибуты > псевдоклассы > теги

Критерии хорошего селектора:

- Уникальность (выбирает только нужные элементы)
- Минимальная сложность
- Устойчивость к изменениям верстки
- Читаемость для других разработчиков
- Пример плохого селектора: `div > div > table > tr > td:nth-child(2)`
- Пример хорошего селектора: `.products td:nth-child(2)`

### Основы XPath

XPath (XML Path Language) — это язык для поиска и навигации по элементам в XML-документе. В тестировании, особенно автоматизированном, XPath часто используется для нахождения и взаимодействия с элементами на веб-страницах. Вот основные вещи, которые нужно знать:
Что такое XPath?


XPath помогает найти элемент в дереве XML или HTML. Поскольку HTML является подмножеством XML, XPath может использоваться для нахождения элементов на веб-странице. Это полезно для тестировщиков при автоматизации тестирования, например, через инструменты вроде Selenium.
Основные синтаксисы XPath

1. Абсолютный XPath — начинается с корневого элемента и идет вниз по всей структуре документа.

`/html/body/div[1]/div[2]/a`

   Такой XPath может быть не очень надежен, так как малейшие изменения в структуре страницы могут его "сломать".

2. Относительный XPath — начинается с текущего узла или текущей позиции.

`//div[@class='example']/a`

   Этот подход более гибкий и надежный, так как не жестко привязан ко всей структуре страницы.

Основные выражения

- `//`: выбирает узлы в документе, соответствующие независимо от их местоположения.

`//input[@id='username']`

- `/`: выбирает непосредственного потомка текущего узла.

`/html/body/div`

- `[@attribute='value']`: выбирает элемент с определённым атрибутом.

`//button[@type='submit']`

- `text()`: выбирает текстовое содержимое элемента.

`//span[text()='Привет']`

- `*`: выбирает все элементы у которых класс button

`//*[@class='button']`

 
Примеры использования XPath

- Поиск элемента по ID:

`//*[@id='login']`

- Поиск элемента по имени класса:

`//div[@class='header']`

- Поиск элемента с определённым текстом:

`//a[text()='Войти']`

- Поиск элемента с атрибутом, начинающимся с определённого текста:

`//input[starts-with(@name, 'user')]`

- Поиск элемента, который содержит (именно содержит а не равен) определённый текст:

`//p[contains(text(), 'подробнее')]`

 
Полезные рекомендации

- Соблюдайте осторожность при использовании абсолютных путей — изменения в структуре страницы могут легко их нарушить.
- Давайте предпочтение относительным путям; это делает тесты более гибкими.
- Используйте уникальные и устойчивые атрибуты (например, id, name) для составления XPath.
- Всегда проверяйте ваши XPath-выражения в браузере (например, инструменты разработчика в Chrome) прежде чем использовать в тестах.

### Оси XPath

Что такое оси XPath?

Оси — это направления навигации в DOM-дереве относительно текущего узла. Они позволяют находить элементы на основе их отношений: родители, дети, соседи и т.д.
Зачем нужны оси?

Когда стандартные локаторы не работают из-за:

- Динамических ID/классов
- Сложной вложенности элементов
- Отсутствия уникальных атрибутов

Основные оси XPath с примерами

Рассмотрим HTML-структуру:

```html
<table id="users">
  <tr>
    <td>Alex</td>
    <td>25</td>
    <td>
      <button class="edit">Edit</button>
    </td>
  </tr>
  <tr>
    <td>Maria</td>
    <td>30</td>
    <td>
      <button class="delete">Delete</button>
    </td>
  </tr>
</table>
```

1. `child:`: - Дочерние элементы

По умолчанию в `//div/span`

Пример: Найти все кнопки в таблице

`//table[@id='users']/child::tr/child::td/child::button`

2. `parent:`: - Родительский элемент

Пример: Найти строку таблицы по кнопке

`//button[text()='Delete']/parent::td/parent::tr`

3. `following-sibling:`: - Следующие братские элементы

Пример: Найти возраст Maria (следующий за именем)

`//td[text()='Maria']/following-sibling::td[1]`

4. `preceding-sibling:`: - Предыдущие братские элементы

Пример: Найти имя по возрасту

`//td[text()='30']/preceding-sibling::td[1]`

5. `ancestor:`: - Предки

Пример: Найти таблицу по кнопке

`//button[@class='delete']/ancestor::table`

6. descendant:: - Потомки

Пример: Найти все кнопки в таблице

`//table[@id='users']/descendant::button`


Практическое задание

Дана структура:

```html
<div class="cart">
  <div class="item" data-id="1">
    <span class="name">Phone</span>
    <span class="price">$500</span>
    <input type="checkbox" class="select">
  </div>
  <div class="item" data-id="2">
    <span class="name">Laptop</span>
    <span class="price">$1200</span>
    <input type="checkbox" class="select">
  </div>
</div>
```

Задачи:

- Найти чекбокс для товара "Laptop"
- Получить цену товара с data-id="1"
- Выбрать все товары дороже $1000

Решения:

```//span[text()='Laptop']/following-sibling::input```

```//div[@data-id='1']/child::span[@class='price']```

```//span[@class='price'][translate(.,'$','')>1000]/ancestor::div```


Оси XPath — мощный инструмент для навигации по DOM. Не пугайтесь если слабо поняли как пользоваться осями. Главное сейчас это получить общее понимание что возможно делать с их помощью. Не стоит их заучивать целенаправленно, на практике в дальнейшем понимание и так прийдет.

Ключевые оси:

- child, parent
- following-sibling, preceding-sibling
- ancestor, descendant

Применяйте когда:

- Элементы не имеют уникальных атрибутов
- Нужна навигация по отношениям элементов

Избегайте:
- Слишком сложных локаторов
- Жёсткой привязки к позициям

Золотое правило:

- "Если можно обойтись без осей лучше их не использовать. Если задача сложная — применяйте оси XPath."

- [Официальная документация XPath](https://developer.mozilla.org/ru/docs/Web/XPath)
- [XPath Tester](https://extendsclass.com/xpath-tester.html)

## Основные методы Selenium WebDriver в Python

Цель: Освоить ключевые методы Selenium для эффективной автоматизации веб-тестирования.

1. Инициализация драйвера и навигация


```python
from selenium import webdriver

# Создание драйвера
driver = webdriver.Chrome()

# Открытие URL
driver.get("https://example.com")

# Навигация
driver.back()      # Назад
driver.forward()   # Вперед
driver.refresh()   # Обновить

# Закрытие
driver.close()     # Текущая вкладка
driver.quit()      # Весь браузер
```

2. Поиск элементов (Finders)

Основные стратегии:

```python
# По ID
element = driver.find_element(By.ID, "username")

# По имени
element = driver.find_element(By.NAME, "email")

# По XPath
element = driver.find_element(By.XPATH, "//button[@type='submit']")

# По CSS-селектору
element = driver.find_element(By.CSS_SELECTOR, ".primary-btn")

# По тексту ссылки
element = driver.find_element(By.LINK_TEXT, "Contact Us")
```

💡 Важно: Используйте find_elements (во множественном числе) для получения списка элементов.

3. Взаимодействие с элементами

Базовые операции:

```python
# Ввод текста
search_field = driver.find_element(By.ID, "search")
search_field.send_keys("Selenium docs")

# Очистка поля
search_field.clear()

# Клик
submit_button = driver.find_element(By.XPATH, "//button")
submit_button.click()

# Получение текста
error_message = driver.find_element(By.CLASS_NAME, "error")
print(error_message.text)  # Выведет "Invalid credentials"

# Получение атрибута
link = driver.find_element(By.LINK_TEXT, "Privacy Policy")
print(link.get_attribute("href"))  # Выведет URL
```

4. Ожидания (Waits)

Неявные (Implicit):

```python
# Установка глобального таймаута (сек)
driver.implicitly_wait(10)
```

Явные (Explicit):

```python
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

wait = WebDriverWait(driver, 15)

# Ожидание появления элемента
element = wait.until(EC.visibility_of_element_located((By.ID, "loader")))

# Ожидание кликабельности
button = wait.until(EC.element_to_be_clickable((By.CSS_SELECTOR, ".submit-btn")))
```

5. Работа с окнами и вкладками

```python
# Текущая вкладка
main_window = driver.current_window_handle

# Открытие новой вкладки
driver.execute_script("window.open('');")

# Переключение между вкладками
for handle in driver.window_handles:
    if handle != main_window:
        driver.switch_to.window(handle)

# Возврат на основную вкладку
driver.switch_to.window(main_window)

# Размер окна
driver.set_window_size(1200, 800)
```

6. Работа с фреймами и алертами

```python
# Переключение на фрейм
driver.switch_to.frame("sidebar-frame")

# Возврат в основной контент
driver.switch_to.default_content()

# Работа с алертами
alert = driver.switch_to.alert
print(alert.text)  # Текст предупреждения
alert.accept()     # Подтвердить
```

## Инструкция assert в Python

Инструкция assert — это ключевой инструмент для написания тестов, который позволяет проверять выполнение условий в вашем коде. Давайте разберём её простыми словами с примерами.
Что такое assert?

assert — это встроенная инструкция Python, которая проверяет истинность выражения:

- Если выражение истинно (True), ничего не происходит, программа продолжает работу
- Если выражение ложно (False), возбуждается исключение AssertionError

Базовый синтаксис:

```python
assert условие, "Сообщение об ошибке"  # Сообщение необязательно
```

Простые примеры
Пример 1: Проверка равенства

```python
def add(a, b):
    return a + b

# Проверяем, что 2 + 2 равно 4
assert add(2, 2) == 4
```

Пример 2: Проверка с сообщением об ошибке

```python
def divide(a, b):
    return a / b

result = divide(10, 2)
assert result == 5, f"Ожидалось 5, получено {result}"
```

Как работает assert в pytest

Pytest расширяет возможности обычного assert, предоставляя подробную информацию при падении теста:
Пример с pytest:

```python
def test_list_operations():
    numbers = [1, 2, 3]
    
    # Проверка включения элемента
    assert 2 in numbers
    
    # Проверка длины списка
    assert len(numbers) == 3
    
    # Проверка равенства списков
    assert numbers == [1, 2, 3]
    
    # Проверка с сообщением
    assert numbers[0] == 1, "Первый элемент должен быть 1"
```

Полезные виды проверок (assertions)
1. Проверки равенства

```python
assert x == y
assert x != y
```

2. Проверки с коллекциями

```python
assert item in collection
assert item not in collection
assert len(collection) == 3
```

3. Проверки исключений

```python
import pytest

def test_divide_by_zero():
    with pytest.raises(ZeroDivisionError):
        divide(10, 0)
```

4. Проверки типов

```python
assert isinstance(x, int)
```

5. Проверки с плавающей точкой (с учётом погрешности)

```python
assert abs(0.1 + 0.2 - 0.3) < 0.0001
```

6. Булевы проверки

```python
assert result is True
assert result is not None
```

Расширенные возможности в pytest

Pytest преобразует выражения в assert для понятных сообщений об ошибках:

```python
def test_complex_assert():
    expected = {"a": 1, "b": 2}
    actual = {"a": 1, "b": 3}
    
    assert actual == expected
```

При падении теста вы увидите подробное сравнение словарей:

```
E       AssertionError: assert {'a': 1, 'b': 3} == {'a': 1, 'b': 2}
E         Differing items:
E         {'b': 3} != {'b': 2}
E         Full diff:
E         - {'a': 1, 'b': 2}
E         + {'a': 1, 'b': 3}
```

Пользовательские сообщения об ошибках

Вы можете добавлять пояснения к проверкам:

```python
def test_with_message():
    result = some_complex_calculation()
    assert result == expected, f"Calculation failed. Expected {expected}, got {result}"
```

Когда использовать assert

- В тестах — для проверки ожидаемого поведения кода
- При отладке — для проверки инвариантов и промежуточных результатов
- В разработке — для проверки входных параметров функций (хотя для production-кода лучше использовать явные проверки с выбрасыванием исключений)

Чего нельзя делать с assert

- Не используйте assert для валидации пользовательского ввода — эти проверки можно отключить при запуске Python с флагом -O
- Не делайте в assert побочных эффектов:

```python
# Плохо:
assert delete_all_files(), "Не удалось удалить файлы"

# Хорошо:
success = delete_all_files()
assert success, "Не удалось удалить файлы"
```

Полезные советы

- Одно утверждение — одна проверка — так проще локализовать ошибку
- Используйте описательные сообщения — особенно для сложных проверок
- Проверяйте не только happy path — тестируйте граничные случаи и ошибочные сценарии
- Используйте специализированные assert-методы для сложных объектов (например, в unittest)

Assert — это ваш главный инструмент для написания тестов. Чем лучше вы освоите различные виды проверок, тем эффективнее будут ваши тесты.

# Pytest

Pytest — фреймворк для тестирования на Python, который помогает создавать и запускать тесты 

Основные возможности:

- Параметризация тестов — запуск одного и того же теста с разными входными данными. ru.wikipedia.org*thecode.media
- Фикстуры — специальные функции, которые помогают подготовить окружение для тестов и убрать тестовые данные после их выполнения.
- Метки — декораторы, которые позволяют корректировать поведение тестов: пропускать их, ожидать определённых результатов, передавать разные входные данные.
- Фильтрация тестов — возможность выбирать, какие тесты запускать, и как они должны себя вести.

### Соглашения об именовании в pytest

1. Имена тестовых функций

- Должны начинаться с test_
pytest автоматически обнаруживает и выполняет такие функции

Пример:
```python

# Правильно - будет найден pytest'ом
def test_addition():
    assert 1 + 1 == 2

# Неправильно - будет проигнорирован
def addition_test():
    assert 1 + 1 == 2
```

2. Имена тестовых классов

- Должны начинаться с Test (с большой буквы)
Содержат тестовые методы, которые начинаются с test_

Пример:
```python

# Правильно - класс будет обнаружен
class TestCalculator:
    def test_add(self):
        assert 2 + 2 == 4
    
    def test_multiply(self):
        assert 3 * 3 == 9

# Неправильно - класс будет проигнорирован
class CalculatorTests:
    def test_add(self):
        assert 2 + 2 == 4
```

3. Почему это важно?

Автоматическое обнаружение: pytest использует эти соглашения для автоматического поиска тестов без необходимости дополнительной конфигурации.

Читаемость: Единый стандарт делает код понятным для всех разработчиков.

Структура: Позволяет организовать тесты логически:

```python
class TestUserRegistration:
    def test_valid_registration(self):
        # тест валидной регистрации
        pass
    
    def test_invalid_email(self):
        # тест невалидного email
        pass
    
    def test_duplicate_username(self):
        # тест дублирующегося username
        pass
```

4. Что происходит при нарушении правил?

- Функции/классы, не следующие соглашениям, игнорируются при запуске тестов
  pytest не считает их тестами и пропускает при выполнении

Эти соглашения являются частью "соглашения вместо конфигурации" (convention over configuration) философии pytest, что делает фреймворк простым в использовании.
 
## Основы параметризации

Параметризация тестов - это мощная фишка pytest, которая позволяет запускать один и тот же тест с разными входными данными. Это избавляет от необходимости писать много почти одинаковых тестов.

Для параметризации используется декоратор @pytest.mark.parametrize. Вот простой пример:

```python
import pytest

# Функция, которую мы тестируем
def add(a, b):
    return a + b

# Параметризованный тест
@pytest.mark.parametrize("a, b, expected", [
    (1, 2, 3),
    (5, 5, 10),
    (0, 0, 0),
    (-1, 1, 0),
])
def test_add(a, b, expected):
    assert add(a, b) == expected
```

В этом примере:

- "a, b, expected" - это названия параметров, которые будут переданы в тест
- Список кортежей - это набор значений для этих параметров
- Тест запустится 4 раза с разными комбинациями параметров

Как это работает

Когда pytest видит декоратор @parametrize, он:

- Берет первый набор параметров (1, 2, 3) и запускает тест с ними
- Берет второй набор (5, 5, 10) и запускает тест снова
- И так для всех наборов

Если тест упадет, pytest покажет, с какими именно параметрами это произошло.

### Параметризация классов

Можно параметризовать весь тестовый класс:

```python
@pytest.mark.parametrize("x, y", [
    (1, 2),
    (3, 4),
])
class TestMath:
    def test_add(self, x, y):
        assert x + y == sum([x, y])
    
    def test_multiply(self, x, y):
        assert x * y == x * y
```

Каждый тест в классе запустится для каждого набора параметров.
Параметризация фикстур

Можно параметризовать фикстуры:

```python
import pytest

@pytest.fixture(params=["apple", "banana", "orange"])
def fruit(request):
    return request.param

def test_fruit(fruit):
    assert fruit in ["apple", "banana", "orange"]
```

Тест запустится 3 раза - по одному для каждого фрукта.
Комбинированная параметризация

Можно использовать несколько декораторов parametrize:

```python
@pytest.mark.parametrize("x", [1, 2])
@pytest.mark.parametrize("y", [10, 20])
def test_combine(x, y):
    print(x, y)
```

Это создаст все возможные комбинации:

- x=1, y=10
- x=1, y=20
- x=2, y=10
- x=2, y=20

Параметризация с ID

Можно задавать понятные имена для каждого набора параметров:

```python
@pytest.mark.parametrize("a, b, expected", [
    pytest.param(1, 2, 3, id="small numbers"),
    pytest.param(100, 200, 300, id="big numbers"),
    pytest.param(0, 0, 0, id="zeros"),
])
def test_with_ids(a, b, expected):
    assert a + b == expected
```

В отчете будут эти имена, что упрощает понимание, где произошла ошибка.
Практические советы

- Используйте параметризацию, когда тестируете одну и ту же логику с разными данными
- Не переусердствуйте - если тесты становятся слишком сложными, лучше разбить их
- Используйте ids для понятных имен тестовых случаев
- Комбинируйте с фикстурами, когда нужно сложное тестовое окружение
- Проверяйте граничные случаи (0, отрицательные числа, пустые строки и т.д.)

Пример из реальной жизни

Допустим, мы тестируем функцию проверки пароля:

```python
def is_valid_password(password):
    return (
        len(password) >= 8 and
        any(c.isupper() for c in password) and
        any(c.isdigit() for c in password)
    )

@pytest.mark.parametrize("password, expected", [
    ("Abc12345", True),       # валидный
    ("abc12345", False),      # нет заглавной
    ("Abcdefgh", False),      # нет цифры
    ("A1b2", False),          # слишком короткий
    ("", False),              # пустой
    ("A!@#$%^&1", True),      # спецсимволы, но есть заглавная и цифра
    ("12345678", False),      # нет заглавной
    ("ABCDEFG1", True),       # только заглавные и цифра
])
def test_password_validation(password, expected):
    assert is_valid_password(password) == expected
```

Этот тест проверяет множество случаев без дублирования кода.

Параметризация делает тесты более компактными и поддерживаемыми, а также помогает проверить больше сценариев с минимальными усилиями.

## Фикстуры (fixtures) в pytest

Фикстуры в pytest — это мощный инструмент для подготовки тестового окружения и очистки после тестов. Они помогают избежать дублирования кода и делают тесты более читаемыми и поддерживаемыми.
Что такое фикстуры?

Фикстуры — это функции, которые выполняются до (а иногда и после) тестов для:

- Подготовки данных
- Настройки среды
- Инициализации ресурсов (базы данных, файлы, сетевые соединения)
- Очистки после тестов

Простейший пример фикстуры

```python
import pytest

# Объявляем фикстуру
@pytest.fixture
def sample_data():
    return [1, 2, 3, 4, 5]

# Используем фикстуру в тесте
def test_sum(sample_data):
    assert sum(sample_data) == 15
```

Здесь:

- sample_data — это фикстура, которая возвращает список чисел
- pytest автоматически передает результат фикстуры в тест как аргумент

Основные возможности фикстур
1. Область видимости (scope)

Фикстуры могут иметь разную область видимости:

```python
@pytest.fixture(scope="function")  # По умолчанию - создается для каждой функции
def func_scope():
    return "function"

@pytest.fixture(scope="class")  # Одна на класс
def class_scope():
    return "class"

@pytest.fixture(scope="module")  # Одна на модуль
def module_scope():
    return "module"

@pytest.fixture(scope="session")  # Одна на все тесты
def session_scope():
    return "session"
```

2. Автоматическая очистка (yield вместо return)

Фикстура может выполнять код после завершения теста:

```python
@pytest.fixture
def db_connection():
    # Подготовка (выполняется до теста)
    conn = create_db_connection()
    yield conn  # передаем соединение в тест
    
    # Очистка (выполняется после теста)
    conn.close()
    print("Connection closed")
```

3. Зависимости между фикстурами

Фикстуры могут зависеть от других фикстур:

```python
@pytest.fixture
def user():
    return {"name": "Alice", "age": 25}

@pytest.fixture
def active_user(user):  # Зависит от фикстуры user
    user["active"] = True
    return user

def test_user_status(active_user):
    assert active_user["active"] is True
```

### Практические примеры

Пример 1: Работа с временными файлами

```python
import pytest
import os

@pytest.fixture
def temp_file(tmp_path):  # tmp_path - встроенная фикстура pytest
    file_path = tmp_path / "test_file.txt"
    with open(file_path, 'w') as f:
        f.write("Hello, pytest!")
    yield file_path
    # Файл автоматически удалится после теста

def test_file_content(temp_file):
    with open(temp_file) as f:
        content = f.read()
    assert content == "Hello, pytest!"
```

Пример 2: Тестирование с базой данных

```python
import pytest
import sqlite3

@pytest.fixture(scope="module")
def db():
    conn = sqlite3.connect(":memory:")
    cursor = conn.cursor()
    cursor.execute("CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)")
    cursor.execute("INSERT INTO users (name) VALUES ('Alice'), ('Bob')")
    conn.commit()
    yield conn
    conn.close()

def test_user_count(db):
    cursor = db.cursor()
    cursor.execute("SELECT COUNT(*) FROM users")
    count = cursor.fetchone()[0]
    assert count == 2

def test_user_names(db):
    cursor = db.cursor()
    cursor.execute("SELECT name FROM users ORDER BY id")
    names = [row[0] for row in cursor.fetchall()]
    assert names == ["Alice", "Bob"]
```

Пример 3: Фикстура с параметризацией

```python
import pytest

@pytest.fixture(params=["user1", "user2", "user3"])
def user(request):
    return {"username": request.param, "active": True}

def test_users(user):
    assert user["active"] is True
    assert user["username"] in ["user1", "user2", "user3"]
```

Встроенные фикстуры pytest

Pytest предоставляет множество полезных встроенных фикстур:

- tmp_path - создает временную директорию
- capsys - для перехвата вывода в stdout/stderr
- monkeypatch - для временного изменения объектов
- request - информация о запущенном тесте

Пример с capsys:

```python
def greet(name):
    print(f"Hello, {name}!")

def test_greet(capsys):
    greet("Alice")
    captured = capsys.readouterr()
    assert captured.out == "Hello, Alice!\n"
```

Советы по работе с фикстурами

- Именуйте фикстуры понятно - имена должны отражать их назначение
- Используйте правильные scope - не используйте широкие scope без необходимости
- Разделяйте фикстуры - лучше несколько маленьких фикстур, чем одна большая
- Используйте yield для очистки - это гарантирует выполнение кода очистки
- Выносите общие фикстуры в conftest.py - для использования в нескольких тестовых файлах

Пример с conftest.py

Если у вас несколько тестовых файлов используют одни и те же фикстуры, вы можете вынести их в файл conftest.py в корне тестовой директории:

```python
# conftest.py
import pytest

@pytest.fixture
def common_data():
    return {"key": "value"}

# test_file.py
def test_data(common_data):  # Фикстура будет автоматически доступна
    assert common_data["key"] == "value"
```

Фикстуры — это одна из самых мощных возможностей pytest. Они помогают создавать поддерживаемые, читаемые и надежные тесты, минимизируя дублирование кода и обеспечивая правильную подготовку и очистку тестового окружения.

## Page Object Model

Что такое Page Object Model?

POM — это дизайн-паттерн, где каждая веб-страница (или её компонент) представляется в виде класса. Логика взаимодействия с элементами страницы инкапсулируется внутри класса, отделяя её от тестовых сценариев.
Основные принципы POM:

- Один класс = одна страница/компонент
- Например: LoginPage, ProfilePage, HeaderComponent.

- Элементы страницы — это атрибуты класса
- Локаторы элементов (XPath, CSS) объявляются в классе.

- Методы = действия пользователя
- Каждый метод класса описывает действие: клик, ввод текста и т.д.

- Изоляция изменений
- Если изменился локатор, правки вносятся только в соответствующий Page Object.

Преимущества POM:

- Удобство поддержки: Изменения в UI затрагивают только Page Object, а не тесты.
- Повторное использование: Методы страницы можно использовать в разных тестах.
- Читаемость: Тесты выглядят как бизнес-сценарии (например, login_page.enter_credentials("user", "pass")).

Практический пример

Сценарий: Автоматизация входа на сайт example.com.
Структура проекта:

```
project/
├── pages/
│   ├── __init__.py
│   └── login_page.py  # Page Object для страницы входа
└── tests/
    └── test_login.py  # Тесты
```

1. Page Object: pages/login_page.py

```python
from selenium.webdriver.common.by import By
from selenium.webdriver.remote.webdriver import WebDriver

class LoginPage:
    # Локаторы элементов
    USERNAME_INPUT = (By.ID, "username")
    PASSWORD_INPUT = (By.NAME, "password")
    SUBMIT_BUTTON = (By.XPATH, "//button[@type='submit']")
    ERROR_MESSAGE = (By.CLASS_NAME, "error")
    
    def __init__(self, driver: WebDriver):
        self.driver = driver
        self.driver.get("https://example.com/login")
    
    # Метод для ввода логина/пароля
    def enter_credentials(self, username: str, password: str) -> None:
        self.driver.find_element(*self.USERNAME_INPUT).send_keys(username)
        self.driver.find_element(*self.PASSWORD_INPUT).send_keys(password)
    
    # Метод для отправки формы
    def submit(self) -> None:
        self.driver.find_element(*self.SUBMIT_BUTTON).click()
    
    # Метод для проверки ошибки
    def get_error_text(self) -> str:
        return self.driver.find_element(*self.ERROR_MESSAGE).text
```

2. Тест: tests/test_login.py

```python
from selenium import webdriver
import pytest
from pages.login_page import LoginPage

@pytest.fixture
def browser():
    driver = webdriver.Chrome()
    yield driver
    driver.quit()

def test_invalid_login(browser):
    # Инициализация Page Object
    login_page = LoginPage(browser)
    
    # Вызов методов страницы
    login_page.enter_credentials("wrong_user", "wrong_pass")
    login_page.submit()
    
    # Проверка результата
    assert "Invalid credentials" in login_page.get_error_text()
```

Объяснение:

- Тест не содержит локаторов или Selenium-команд напрямую.
- Логика страницы изолирована в LoginPage.
- При изменении локаторов (например, USERNAME_INPUT) правки вносятся только в login_page.py.

Зачем нужна эта "сложность" с классами?

Без POM:

- Если разработчик изменит кнопку "Войти", тебе придется исправить 15 тестов, где эта кнопка используется.
- Тесты — это сплошной код с find_element и send_keys, в котором сложно разобраться.

С POM:

- Ты меняешь локатор кнопки в одном месте — в классе LoginPage.
- Все 15 тестов автоматически начинают работать с новой кнопкой.
- Тесты читаются как обычные сценарии, а не код.

### Разберем импорты (import, from)

Это частая "боль" новичков. Давай разберем твой пример.

В файле tests/test_login.py есть строка:

```python
from pages.login_page import LoginPage
```

"Переведем" ее на русский:

- `from pages.login_page` — означает "из файла login_page.py, который находится в папке pages..."
- `import LoginPage` — "...возьми и дай мне доступ к классу LoginPage".

Как это работает шаг за шагом:

- pages — это папка. Файл __init__.py внутри нее делает ее Python-пакетом. Теперь Python понимает, что pages — это не просто папка, а место, откуда можно импортировать код.
- login_page — это имя файла без расширения .py.
- LoginPage — это имя класса, который объявлен внутри файла login_page.py.

Альтернативный способ импорта (чтобы было понятнее):

```python
# test_login.py (альтернативный вариант)
from pages import login_page # Импортируем весь файл-модуль

def test_invalid_login(browser):
    login_page = login_page.LoginPage(browser) # Обращаемся к классу через точку
    ...
```

В этом варианте мы импортируем не класс, а целый файл (login_page). Тогда чтобы создать объект, нам нужно указать login_page.LoginPage().

Первый способ (from ... import ...) считается более чистым и читаемым.

### Что происходит при создании объекта:

```python
# Когда мы пишем:
page = LoginPage(driver)

# Python выполняет:
# 1. Создает новый объект класса LoginPage
# 2. Вызывает метод __init__(self, driver)
# 3. Сохраняет driver в self.driver
# 4. Создает атрибуты с локаторами
```
Что происходит при вызове метода:

```python
# Когда мы пишем:
page.login("user123", "pass123")

# Python выполняет:
# 1. Ищет метод login в классе LoginPage
# 2. Вызывает login(self, "user123", "pass123")
# 3. self автоматически становится объектом page
# 4. Выполняются все действия внутри метода
```

### Что такое self?

self - это ссылка на конкретный объект, с которым мы работаем в данный момент.

self - это слово, которое означает:

- "эта страница"
- "этот объект"
- "конкретный экземпляр"

Когда вы видите self.driver - думайте "драйвер этой страницы"
Когда вы видите self.username_field - думайте "поле username этой страницы"

Без self Python не понимал бы, с каким именно объектом работать!

Подведем итоги:

- Page Object (pages/login_page.py) — это ассистент для конкретной страницы сайта.
- Тест (tests/test_login.py) — это сценарий, который обращается к ассистентам: "Ассистент страницы логина, сделай то-то. Ассистент главной страницы, проверь то-то".
- Локаторы — это глаза ассистента, которыми он находит кнопки и поля на странице.
- Импорты — это способ позвать нужного ассистента в свой тест.

Этот подход требует чуть больше кода вначале, но он ОЧЕНЬ быстро окупается, когда проект растет, а требования к интерфейсу постоянно меняются.

# Файл requirements.txt

В основе проектов лежат библиотеки и другие зависимости, и процесс установки каждой из можно упростить. 

Файл requirements.txt – файл, содержащий список пакетов и библиотек, которые необходимы для работы с проектом и которые можно установить вместе с этим файлом. Это обеспечивает тот факт, что ваша среда будет согласованной, а совместная работа непринужденной. 

Файл requirements.txt для проекта на Python можно создать вручную или с помощью команды pip freeze. 

Команда pip freeze формирует список всех установленных в текущем Python-окружении пакетов вместе с их версиями. Чтобы сохранить этот список в файл, нужно использовать перенаправление вывода:

pip freeze > requirements.txt

Недостаток: pip freeze выводит абсолютно все пакеты из окружения, включая те, которые не используются непосредственно в проекте.
Для более продвинутого использования можно применять фильтрацию результатов pip freeze, например, если нужно исключить определённые пакеты:

pip freeze | grep -v "package-to-exclude" > requirements.txt

# Allure

​Allure — это удобный инструмент для создания отчетов по тестам. Он показывает результаты в понятном виде, помогает анализировать ошибки и улучшать качество кода. Работает с разными языками программирования и фреймворками, поэтому подходит почти для любого проекта.
Что умеет Allure?

- Наглядные отчеты – графики, диаграммы и детальная информация по каждому тесту.
- Поддержка разных языков – Java, Python, JavaScript, C# и другие.
- Интеграция с CI/CD – автоматическая генерация отчетов в Jenkins, GitLab CI и других системах.
- Разные виды данных в отчетах – шаги тестов, скриншоты, логи, ошибки и многое другое.

Почему Allure полезен?

- Показывает, какие тесты прошли, а какие упали и почему.
- Помогает быстро находить и чинить баги.
- Упрощает работу в больших проектах с сотнями тестов.

Итог: Allure делает тестирование прозрачным, а поиск проблем — быстрым. Это must-have инструмент для автоматизаторов! 

Практическое задание

Задача: Написать тест проверки поиска на странице Google с Allure-интеграцией
Требования:

- Тест переходит на https://google.com
- Вводит запрос "Allure Framework"
- Проверяет наличие результатов поиска
- При падении теста добавляет в отчёт:
- Скриншот
- Текущий URL
- Заголовок страницы

Решение:

```python

import allure 
from selenium import webdriver 
from selenium.webdriver.common.by import By 

@allure.suite("Поиск в Google") 
@allure.title("Поиск информации об Allure Framework") 
def test_google_search(): 
    driver = webdriver.Chrome() 
    driver.get("https://google.com") 

    try: 
        with allure.step("Ввод поискового запроса"): 
            search_box = driver.find_element(By.NAME, "q") 
            search_box.send_keys("Allure Framework") 
            search_box.submit() 

        with allure.step("Проверка результатов"): 
            results = driver.find_elements(By.CSS_SELECTOR, "div.g") 
            assert len(results) > 0, "Нет результатов поиска" 

    except Exception as e: 
        # Добавляем вложения при ошибке 
        allure.attach( 
            driver.get_screenshot_as_png(), 
            name="search_error", 
            attachment_type=allure.attachment_type.PNG 
        ) 
        allure.attach( 
            f"URL: {driver.current_url}\nTitle: {driver.title}", 
            name="page_info", 
            attachment_type=allure.attachment_type.TEXT 
        ) 
        raise e 
    finally: 
        driver.quit()
```

- Сгенерируйте отчёт: allure serve ./allure-results
